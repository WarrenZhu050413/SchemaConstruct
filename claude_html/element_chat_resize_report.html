üìâ Failure Analysis — Pre-fix (commit 3315f1b, October 3, 2025) ‚ñ∂
Observed issues
  • Shadow host wrapper enforced pointer-events: none, while Rnd wrapper lacked an override, so drag-resize handles never received the mousedown event (see retrospective note in claude_html/element_chat_ui_plan.html:373).
  • UI offered no visual cue; default react-rnd handles are 1px edges. Without explicit grips, QA reported resize as "non-functional" (claude_html/element_chat_ui_plan.html:374-382).
  • Only onResizeStop persisted new dimensions (src/components/ElementChatWindow.tsx @ commit 3315f1b:510-517), so during an in-progress drag the frame snapped back because state stayed stale.
Impact
  • Users could not reliably change width/height — resizing appeared broken despite configuration being present.
  • Storage persisted defaults, leading reopened chats to ignore any manual drag adjustments.
‚öôÔ∏è Current Implementation — Post-fix (commit 6b523e7, October 5, 2025) ‚ñ∂
Key changes
  • Explicitly restores pointer events on the interactive frame (style={{ pointerEvents: 'auto' }}) so handles rise above the pass-through host — src/components/ElementChatWindow.tsx:1257-1281.
  • Adds live handleResize updates to windowSize, eliminating snap-back and keeping layout responsive while dragging — src/components/ElementChatWindow.tsx:1030-1038.
  • Persists anchor-relative offset on resize/drag to prevent post-resize reposition drift when the page scrolls — src/components/ElementChatWindow.tsx:1006-1028.
  • Introduces custom high-contrast grips injected via resizeHandleComponent for all active edges/corners — src/components/ElementChatWindow.tsx:2247-2276.
Persistence
  • Updated saveSession pipeline reuses refs so new size values captured during drag land in storage without race conditions — src/components/ElementChatWindow.tsx:520-590.
