State Pseudo-classes

  • UI & interaction: `:hover`, `:focus`, `:active`, `:visited`
  • Form state: `:checked`, `:required`, `:disabled`, `:valid`
  • Negation: `:not(.disabled)` filter out matches
üìç Combinators & Relationships ‚ñ∂
  • Descendant: `.card .title` (space) for any nested descendant
  • Child: `.card > .title` immediate child only
  • Adjacent sibling: `.button + .hint` next sibling
  • General sibling: `.alert ~ p` later siblings

Grouping & Lists

Comma-separated selectors apply the same rules to multiple targets: `.btn, .link { ... }`

üìä Structural Pseudo-classes ‚ñ∂

Positional Filters

  • `:first-child`, `:last-child`, `:only-child`
  • `:nth-child(an+b)` family including `:nth-of-type` variants
  • `:first-of-type`, `:last-of-type` for element type-aware matching

Content-aware Selectors

  • `:empty` for elements without children
  • `:has(selector)` (CSS4, limited support) selects parent containing match
  • `:is()`, `:where()` simplify complex lists; `:where()` zero specificity
üìö Pseudo-elements & Generated Content ‚ñ∂
  • `::before`, `::after` inject decorative content
  • `::first-line`, `::first-letter` typographic control
  • `::selection` highlight color, `::marker` list bullets
  • Form-specific: `::placeholder`, `::file-selector-button`

Pseudo-elements always attach to a base selector: `.alert::before`.

üß≠ Advanced Selector Engines ‚ñ∂

XPath

  • XML-like path syntax: `//div[@role='tab']`
  • Powerful for relational queries (`ancestor::`, `following-sibling::`)
  • Tooling: Selenium, backend parsers

JavaScript APIs

  • `document.querySelector(css)` / `querySelectorAll` (CSS-powered)
  • `getElementById`, `getElementsByClassName`, `matches()`

Automation Locator Engines

  • Playwright: CSS, text, role (`page.getByRole('button', { name: 'Save' })`), test id (`getByTestId`), XPath, custom engines
  • Puppeteer/WebDriver: similar mix, sometimes less ergonomic
üß± Selector Strategy & Resilience Tips ‚ñ∂
  • Prefer semantics: meaningful classes or data attributes (`data-testid`) over brittle DOM chains
  • Limit specificity: avoid overly long selectors; easier overrides
  • Shadow DOM: use host-level hooks, `:host`, or automation-specific shadow piercing APIs
  • Accessibility: role-based selectors align with ARIA landmarks
  • Testing: dedicated test ids keep UI and test surfaces decoupled
  • Performance: right-to-left matching‚Äîkeep rightmost selector simple
