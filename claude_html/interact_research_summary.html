üéØ Core Vision

INTERACT is a generative overview-detail interface that enables non-linear information seeking, spatial knowledge construction, and element-level interaction with AI-generated content through tight coupling of outputs and inputs.

1Ô∏è‚É£ Core Design Philosophy ‚ñ∂

üí° Key Principle: Output as Input

Every output becomes a manipulable input for further exploration. This "tight coupling" maintains cognitive flow and enables progressive refinement without context switching.

Foundation: Shneiderman (1994) - "Visual Information Seeking: Tight Coupling of Dynamic Query Filters with Starfield Displays"

Human-Centric Principles

  • Human beings at the center of all interaction
  • Support personal digital sovereignty
  • Enable users to build their own computational environments
  • Embrace polysemanticity - attach tools anywhere

Web Malleability

  • Web inherently malleable (HTML/CSS/JS client-side)
  • Documents transmitted to browser can be modified
  • Use simplest stacks for compatibility
  • Enable easy customization for everything
Direct Manipulation Principles (Shneiderman) ‚ñ∂
  • Visual representation of the world of action (objects + actions)
  • Rapid, incremental, reversible actions
  • Selection by pointing (not typing)
  • Immediate and continuous display of results
  • Progressive refinement of search parameters
  • Details on demand - heart of hypermedia
üìÑ Source: Visual Information Seeking (CHI 1994) - https://dl.acm.org/doi/10.1145/191666.191775
Non-linearity & Spatial Thinking ‚ñ∂
  • Information seeking is highly non-linear with parallel subprocesses
  • Colocation principle: physically arrange related info spatially
  • Distance = conceptual distance
  • Support hierarchical discovery even when not all info found
  • Spatial hypertext: relationships through arrangement (not explicit links)

üåü Key Insight

Destination content should be viewable without leaving source context. This maintains cognitive flow and prevents disruption of parallel processes.

üîç Three Levels of Engagement

Information Seeking is Non-Linear ‚ñ∂
Information Seeking in Electronic Environments
Information seeking highly non-linear, involving many subprocesses that may jump to each other.
Book

Parallel Subprocess Activation:

  • Continuous Background: Define Problem, Recognize Problem, Reflect stay active throughout
  • Opportunistic Branching: Extract from one source while searching in another tab
  • Cognitive Parallelism: Pattern recognition while reading, associative thinking
  • Interleaved Processing: Rapid switching (Examine + Define + Reflect)

Key Insights:

  • 8 subprocesses with three transition types (Default, High Prob, Low Prob)
  • Define Problem and Examine Results = critical hub nodes
  • Electronic environments enable more parallelism (tabs, windows, copy/paste)
  • Interruptions costly - disrupt multiple parallel processes simultaneously
Exploratory Search (Marchionini 2006) ‚ñ∂
Exploratory Search: From Finding to Understanding
Paper

Lookup Search

  • Discrete, well-structured problems
  • Fact-finding, navigation, Q&A
  • Optimize for precision

Exploratory Search

  • Ill-structured problems
  • Requires LEARNING or INVESTIGATION
  • Maximize recall over precision
  • Combines querying + browsing
Challenge: Maximize recall ‚Üí many results ‚Üí overwhelming
INTERACT Solution: Progressive disclosure, overview-detail, filtering
"Browsing is an exploratory, information-seeking strategy that depends on serendipity. End users rationalize inefficient information-seeking strategies by hoping that incidental learning will have a beneficial cumulative effect. This is a perfect description of what I want!"
Information Foraging Theory ‚ñ∂
Information Foraging Theory (Pirolli & Card 1999)
Animals rely on scents to find prey; humans rely on "information scent" to find information. Sensemaking = iterative foraging + synthesis loops.
PDF

Key Concepts:

  • Information patches: Clustered information sources
  • Information diet: Pursuing one source over another
  • Foraging loop: Seeking and gathering information
  • Synthesis loop: Making sense of gathered info
Output-as-input crucial for maintaining scent during exploration!
5Ô∏è‚É£ Interaction Patterns & UI Paradigms ‚ñ∂
DirectGPT - Direct Manipulation for LLMs (CHI 2024) ‚ñ∂
DirectGPT - https://arxiv.org/pdf/2310.03691
Participants 50% faster, used 50% fewer and 72% shorter prompts vs ChatGPT

Key Principles:

  • Continuous representation of objects - outputs are manipulable
  • Reuse prompt syntax in toolbar - operations become tools
  • Manipulable outputs to control prompts - decouple action from object
  • Undo mechanisms - reversible operations essential
"The difference between my work and DirectGPT is that DirectGPT is more about manipulation and editing. Here, we are thinking about learning rather than creating, about absorbing information rather than producing."

Ideas from DirectGPT:

  • Rapid Operations Through Prompts Reused as Tools: As soon as a prompt finishes, it's added as a button in toolbar
  • Reversible Operations: Undo mechanisms - something we need in INTERACT
  • Highlight changes: User knows when things changed (related to reversibility)
Deictic Gestures & Multimodal Interaction ‚ñ∂
Combining Deictic Gestures and Natural Language for Referent Identification
Spatial deixis is valuable source of information for identifying referents in dialog systems.

Three Advantages of Pointing Gestures:

  • Save generation/analysis of complex referential descriptions
  • Allow reference when linguistic description simply not possible (dozen similar objects)
  • Permit speaker to be vague, imprecise, or ambiguous - use everyday terms instead of precise technical terms

Related Work Using Deictic Interaction:

  • FusAIn: Pen-based interaction for composing generative AI visual prompts
  • Brickify: Direct manipulation on design tokens - colors/shapes hard to describe in language
  • InterChat: Visual analytics using multimodal interactions - drawing selection boxes more intuitive than prompts
  • Stylette: Click components + natural language to change website styles
"Language-only interactions often fall short in precisely conveying patterns. Describing a fluctuation in a time-series chart or clusters in a scatterplot can be cumbersome and prone to errors."

Malleable Overview-Detail Interfaces (2024)
Overview-detail pattern ubiquitous in information systems. Users need representations that help organize/preview large numbers of entities and quickly determine relevance.

Why Overview-Detail for Information Tasks:

Feedforward Design & Turn-Taking Challenges ‚ñ∂

Problem: Turn-Taking Bottleneck

  • Users unwilling to provide relevance feedback in classic turn-taking model
  • Frustration with inability to overlap/respond during LLM generation
  • Want to see outline before detail (errors compound)

INTERACT Solution:

  • Decouple question-asking from answer-getting (at least for now)
  • Allow text-based overlap in human-LLM interactions
  • Feedforward design: show HTML outline first before full detail
  • Buttons for "explain further", "search related content" (user-defined prompts)
Beyond Turn-taking: Text-based Overlap in Human-LLM Interactions
Current LLMs don't allow overlap/response during generation - frustrating for users who want to engage in real-time.
Feedforward in Generative AI
Want to see outline before having more detail because errors compound. Same for seeing HTML outline first.

7Ô∏è‚É£ Net Generation Learning Preferences ‚ñ∂
Digital Natives, Digital Immigrants & Is It Age or IT: Net Generation
EDUCAUSE Research

Characteristics of Digital Natives:

Cognitive Patterns

  • Bricolage over linear: "Hypertext minds, they leap around"
  • Inductive discovery: Observe, hypothesize, figure out rules
  • Parallel processing and multitasking
  • Random access (like hypertext) over sequential

Preferences

  • Image-rich over text-heavy (refuse long text assignments)
  • Graphics before text
  • Experiential learning: Doing vs talking
  • Immediacy: "My info now, not later"
  • Interactive and engaging, not passive
"Learning is participatory; knowing depends on practice and participation. Digital resources enable experiential learning. Rather than being told, Net Geners would rather construct their own learning, assembling information, tools, and frameworks from a variety of sources."

Implications for INTERACT:

  • Support non-linear exploration and jumping
  • Visual/spatial organization over text-heavy displays
  • Interactive and engaging, not passive consumption
  • Allow constructing own learning from varied sources
  • Provide immediate feedback and frequent rewards
"When instructions altered from text-based step-by-step to graphic layout, refusals to do assignment dropped and post-test scores increased."

9Ô∏è‚É£ Implementation Concepts & Features ‚ñ∂

üé® Treat Output as Living Canvas

Unlike typical LLMs where revisiting output is hard, treat output as common ground/canvas for editing. Highlight changes so user knows when things changed.

Key Features to Implement:

Spatial Organization & Cards ‚ñ∂
  • Principle of colocation: Things arranged spatially, relevant info next to each other
  • Reorganize by moving: Drag to rearrange information spatially
  • Card-based architecture:
    • Each card fully self-contained
    • Multiple export formats (JSON/MD/HTML)
    • Unique IDs for tracking/combination
    • Metadata (tags, timestamps, connections)
  • Card combination: Drag onto another to combine (LLM generates fusion)
Export & Modularity ‚ñ∂
Principle: Every single HTML element should be able to be natively exported as JSON or as markdown.
  • Every HTML element has data-export attribute
  • Right-click any element ‚Üí Export as...
  • Batch export selected cards
  • Maintain relationships in export
Interactive Feedback & Relevance ‚ñ∂
"Relevance feedback is powerful way to improve retrieval. However, users often unwilling to take added step in classic turn-taking model. Maybe we need buttons and special gestures for 'explain further', 'search related content', etc. These can even be user-defined prompts and buttons."
  • Expanded chat window with action buttons
  • "Explain further", "Search related content" buttons
  • User-defined prompts and buttons
  • Reversible operations with undo
  • Highlight changes visually
LLM Integration & Conversations ‚ñ∂
  • LLM conversations ‚Üí Nabokov cards
  • Cards can merge, interact, become parts of other cards
  • Prompts reused as toolbar buttons (DirectGPT principle)
  • Grounding in real-world events (element-level interaction)
Technical Stack Considerations ‚ñ∂
"The problem of compatibility to any environment can be solved by using the simplest stacks: HTML, Javascript, and (potentially) REACT."
  • React with drag-and-drop libraries
  • Canvas/SVG for connection visualization

  • GPT-4/Claude for combinations and generation
  • Electron for desktop app (optional)

üéØ INTERACT: Core Vision Summary

What INTERACT Is:

A generative overview-detail interface for non-linear information seeking that bridges lookup ‚Üí learn ‚Üí investigate.

Ten Foundational Principles:

  • 1. Leverages web malleability (HTML/CSS/JS) for user sovereignty and customization
  • 2. Implements tight coupling (output-as-input) to maintain cognitive flow and enable progressive refinement
  • 3. Uses spatial arrangement as cognitive scaffolding and implicit relationship representation
  • 4. Supports non-linear information seeking with progressive disclosure and overview-detail patterns
  • 5. Combines deictic interaction (pointing, clicking elements) with natural language to overcome language limitations
  • 6. Maintains information coexistence (preview without leaving context) to prevent disruptive context switching
  • 7. Enables epistemic actions (interactions that improve cognition, not just achieve goals)
  • 8. Bridges three levels - lookup (databases) ‚Üí learn (specialized software) ‚Üí investigate (INTERACT's focus)
  • 9. Embraces bricolage and hypertext minds of digital natives who leap around, prefer graphics, construct own learning
  • 10. Treats outputs as living canvas for manipulation, with every HTML element exportable and cards as modular building blocks
"The goal is not to use LLMs only, but to have ways that enable us to more fully investigate and synthesize different forms of information together as quickly as possible, supporting exploratory information-seeking strategies that depend on serendipity and incidental learning with beneficial cumulative effects."

Key Differentiators from Existing Work:

  • vs DirectGPT: Focus on learning/absorbing rather than creating/producing
  • vs Malleable Overview-Detail: Embeds chats into UI for dynamic detail specification (not just static customization)
  • vs Traditional LLMs: Outputs are living canvases for manipulation (not just text streams)
  • vs Document Editing Tools: Spatial knowledge construction for investigation (not just linear document creation)
