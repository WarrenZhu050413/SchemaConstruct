Why Element Chat Showed Duplicate Bubbles

Prior to the fix, a combination of the send guard and streaming handling allowed the same payload to be processed twice. The guard only blocked repeats while a send was in flight; once the async call finished, a rerender or double-click could enqueue the same text again. Meanwhile, the streaming consumer appended every assistant chunk even if the content matched the previous message.

Key Observations

  • The signature guard accepted duplicates if the first send had already completed.
  • Streaming responses were always appended, so identical completions became extra assistant bubbles.
  • Result: users saw repeated “You” or “Assistant” messages for one logical turn.
