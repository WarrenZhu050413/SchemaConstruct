innerHTML vs outerHTML: What's the Difference?

Quick Answer

innerHTML: Gets/sets the HTML content inside an element (its children)

outerHTML: Gets/sets the HTML content including the element itself

Visual Comparison

innerHTML

What it touches: Only the CHILDREN

<div id="example"> <p>My name is Joe</p> </div>

Reading:

element.innerHTML
// Returns: "<p>My name is Joe</p>"

Writing:

element.innerHTML = "<span>New content</span>";
// Result:
<div id="example">
  <span>New content</span>
</div>
‚úÖ The <div> stays, only its children change

outerHTML

What it touches: The WHOLE ELEMENT

<div id="example"> <p>My name is Joe</p> </div>

Reading:

element.outerHTML
// Returns: "<div id=\"example\"><p>My name is Joe</p></div>"

Writing:

element.outerHTML = "<span>Replaced</span>";
// Result:
<span>Replaced</span>
‚ö†Ô∏è The <div> is completely replaced!

Side-by-Side Code Examples

innerHTML Example

const div = document.querySelector('#myDiv');

// Before:
// <div id="myDiv">
//   <p>Hello</p>
// </div>

div.innerHTML = '<strong>New</strong>';

// After:
// <div id="myDiv">
//   <strong>New</strong>
// </div>

Note: The div element and its id="myDiv" are preserved.

outerHTML Example

const div = document.querySelector('#myDiv');

// Before:
// <div id="myDiv">
//   <p>Hello</p>
// </div>

div.outerHTML = '<section>New</section>';

// After:
// <section>New</section>

Note: The entire div is gone, replaced by section. The id="myDiv" is lost!

Key Differences

1. Scope of Operation
  • innerHTML: Works inside the element (like editing a folder's contents)
  • outerHTML: Replaces the entire element (like replacing the folder itself)
2. Element Attributes Preservation
  • innerHTML: ‚úÖ Preserves the element's attributes (id, class, data-*, etc.)
  • outerHTML: ‚ùå Loses all attributes unless you include them in the new HTML string
3. Variable Reference After Assignment
  • innerHTML: ‚úÖ The element variable still points to the same element
  • outerHTML: ‚ö†Ô∏è The element variable now points to a removed element (detached from DOM)

Common Use Cases

When to Use innerHTML

  • Updating content inside a container
  • Clearing all children: elem.innerHTML = ''
  • Appending new children
  • Keeping the wrapper element intact
  • Preserving element attributes and event listeners on the parent

When to Use outerHTML

  • Replacing an element with a different tag
  • Completely swapping out an element
  • Reading the full HTML representation of an element
  • Less common in practice (usually replaceWith() is safer)

Security Warning (Both Methods)

‚ö†Ô∏è XSS Risk: Both innerHTML and outerHTML are vulnerable to Cross-Site Scripting (XSS) attacks if you insert untrusted user input.

Dangerous example:

// DON'T DO THIS with user input!
const userInput = "<img src='x' onerror='alert(1)'>";
element.innerHTML = userInput;  // Executes malicious code!

Safe alternatives:

  • Use textContent for plain text (no HTML parsing)
  • Use TrustedHTML objects with sanitization libraries like DOMPurify
  • Create elements with document.createElement() and set properties directly

Practical Example: The Gotcha with outerHTML

const box = document.querySelector('#box');

console.log(box);  // <div id="box">...</div>

box.outerHTML = '<span>New element</span>';

console.log(box);  // Still <div id="box">...</div> (!!!)
console.log(box.parentNode);  // null (detached from DOM!)

// The 'box' variable now points to a removed element!
// The actual DOM has <span>New element</span> in that location
Important: After using outerHTML, your variable still references the old, removed element. You need to query for the new element if you want to continue working with it.

Memory Aid

Think of it this way:

  • innerHTML = "inner content" (what's inside the tags)
  • outerHTML = "outer wrapper included" (the whole thing)
