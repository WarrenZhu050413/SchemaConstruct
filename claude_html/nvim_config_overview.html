‚ö° Telescope Preview Adjustment‚ñ∂

Recommended horizontal layout config (global):

-- ~/.config/nvim/lua/plugins/telescope.lua
opts = {
  defaults = {
    layout_strategy = 'horizontal',
    layout_config = {
      width = 0.96,
      height = 0.92,
      preview_width = 0.68,      -- global default for most pickers
      horizontal = { preview_width = 0.68, results_width = 0.32 },
      mirror = false,
    },
  },
  pickers = { find_files = { hidden = true } },
}

Add the block inside the existing opts table, keeping pickers.find_files as-is. Width values accept 0-1 floats (ratio) or absolute columns. After saving, run :source % inside the file or restart Neovim.

Optional keymap-only override:

map('n', 'sg', function()
  tb.live_grep({ layout_strategy = 'horizontal', layout_config = { horizontal = { preview_width = 0.7 } } })
end, { desc = 'Search in files (grep)' })

Place this in config/keymaps.lua replacing the existing mapping if you want preview tweaks just for live grep.

üìç Config Topology‚ñ∂

Bootstrapping

init.lua sets mapleader to space, boots lazy.nvim, loads config/options.lua, then imports plugin specs from lua/plugins/, and finally applies config/keymaps.lua & config/autocmds.lua.

Core Settings

config/options.lua mirrors VS Code defaults (relative numbers, clipboard integration, split behavior). Tree-sitter folding enabled by default.

Keymaps & Autocmds

config/keymaps.lua centralizes leader maps; config/autocmds.lua handles auto-reload, Markdown tweaks, LSP attach mappings/inlay hints, and quit cleanup.

Reference Cheatsheet

QUICK_COMMANDS.txt in the config root documents the VS Code-style shortcuts for fast lookup.

üèõÔ∏è Plugin Groups‚ñ∂

UI & UX

plugins/ui.lua (Catppuccin, Lualine, Which-key). Transparent toggle via <leader>tt in config/keymaps.lua.

Search & Files

plugins/telescope.lua for fuzzy find/live grep; plugins/explorer_terminal.lua for NvimTree and ToggleTerm.

LSP & Completion

plugins/lsp.lua uses vim.lsp.config (0.11+) with Mason autoinstall; plugins/cmp.lua wires nvim-cmp + LuaSnip.

Formatting & Linting

plugins/format_lint.lua sets Conform format-on-save and nvim-lint on write/leave insert; Mason Tool Installer ensures prettierd, ruff, etc.

Git & Diagnostics

plugins/git_trouble.lua registers Gitsigns with blame preview and Trouble for a Problems pane (<leader>xx).

Markdown Toolkit

plugins/markdown.lua bundles browser preview, inline render, and TOC generation (<leader>mp, <leader>mt).

Sessions

plugins/session.lua auto-saves/restores project sessions and exposes <leader>w* bindings.

Editing Ergonomics

plugins/editing.lua installs autopairs, comment toggle, and surround for VS Code parity.

üìö Navigation Tips‚ñ∂
  • Jump to config: From project buffers run :e ~/.config/nvim/lua/plugins/telescope.lua or use <leader>sf to fuzzy-find telescope.lua.
  • Check effective keys: <leader>sk opens Telescope keymap picker for live lookup.
  • Validate layout changes: After editing, call :Telescope live_grep directly to verify preview width before updating the keymap.
  • Session hygiene: Toggle auto-save with <leader>wa when testing config tweaks to avoid persisting experimental window layouts.
  • Diagnostics view: Use <leader>xx (Trouble) for a VS Code Problems panel; double-press q to close both Trouble and the last buffer thanks to QuitPre autocmd cleanup.
  • Markdown workflows: Inline render is automatic; use <leader>mp only when you need a browser preview.
Peek ~/.config/nvim/QUICK_COMMANDS.txt for the full VS Code-style cheatsheet whenever you forget a binding.
